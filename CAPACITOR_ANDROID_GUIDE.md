# Capacitor Android 앱 빌드 가이드
> MyBob 프로젝트 기준 / 2026-05-22 작성

---

## 1. Capacitor란?

웹 앱(Next.js, React 등)을 **네이티브 앱처럼 감싸주는 도구**입니다.

```
[사용자] → [Android 앱] → [WebView] → [Vercel 웹사이트]
```

- 앱을 열면 안드로이드 WebView(크롬 엔진)가 Vercel URL을 불러옵니다.
- 사용자 입장에서는 일반 앱처럼 보이지만, 실제로는 웹사이트를 앱 껍데기로 감싼 구조입니다.
- API, 서버, DB는 기존 Vercel + Supabase 그대로 사용합니다.

### 웹뷰 방식 vs 정적 빌드 방식

| 구분 | 웹뷰 방식 (이번에 선택) | 정적 빌드 방식 |
|------|------------------------|---------------|
| 구조 | Vercel URL을 앱이 열어줌 | 프론트엔드를 앱에 번들링 |
| 작업량 | 적음 (하루 이내) | 많음 (2~3일, API 분리 필요) |
| 인터넷 | 필수 | 오프라인 일부 가능 |
| Play Store | 통과 가능 | 통과 가능 |
| 적합한 경우 | 빠른 출시, 기능 개발 중 | 완성된 앱, 오프라인 지원 필요 시 |

## 2. 사전 준비

### 필요한 것
- Node.js / npm
- Android Studio (16버전 이상)
- JDK 21 이상 (Java Development Kit)
- Android SDK

### Android Studio 설치 후 환경변수 설정

**시스템 환경 변수** (윈도우 검색 → "시스템 환경 변수 편집"):

1. **ANDROID_HOME** 새로 만들기
   ```
   변수 이름: ANDROID_HOME
   변수 값:   C:\Users\{사용자명}\AppData\Local\Android\Sdk
   ```

2. **Path** 편집 → 새로 만들기
   ```
   %ANDROID_HOME%\platform-tools
   ```

3. **JAVA_HOME** 확인/수정
   ```
   변수 이름: JAVA_HOME
   변수 값:   C:\Program Files\Eclipse Adoptium\jdk-25.0.3.9-hotspot
   ```
   (설치된 JDK 버전에 맞게 경로 조정)

### 환경변수 적용 확인 (새 터미널에서)
```bash
adb --version       # Android 디버그 브릿지
java -version       # Java 버전 (21 이상이어야 함)
```

## 3. Capacitor 설치 및 설정

### 패키지 설치
```bash
npm install @capacitor/core @capacitor/cli @capacitor/android
```

### capacitor.config.ts 생성 (프로젝트 루트)
```typescript
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.mybob.app',        // 앱 고유 ID (패키지명, 나중에 변경 어려움)
  appName: '뭐먹었어',            // 앱 이름
  webDir: 'out',                 // 정적 빌드 폴더 (웹뷰 방식에서는 사용 안 함)
  server: {
    url: 'https://mybob-gamma.vercel.app',  // 실제 Vercel URL
    cleartext: false,
  },
  android: {
    allowMixedContent: false,
    captureInput: true,
    webContentsDebuggingEnabled: false,
  },
};

export default config;
```

> **appId 주의:** Play Store에 한 번 등록하면 변경 불가. 신중하게 결정할 것.

### .gitignore에 android/ 추가
```
# Capacitor
/android
/ios
```

android/ 폴더는 `cap add android` 명령으로 자동 생성되므로 git에 포함할 필요 없음.

## 4. Android 프로젝트 생성

### 순서대로 실행

```bash
# 1. 프로젝트 루트로 이동 (중요!)
cd C:\Users\lucky\Desktop\MyBob

# 2. Android 플랫폼 추가 (android/ 폴더 자동 생성)
cap add android

# 3. 동기화 (설정 파일들을 android/ 폴더로 복사)
cap sync android

# 4. Android Studio로 열기
cap open android
```

> **주의:** `cap add android`는 반드시 `package.json`이 있는 프로젝트 루트에서 실행해야 함.
> 다른 폴더에서 실행하면 "Could not find npm package" 오류 발생.

## 7. 발생했던 오류 및 해결방법

### 오류 1: `npm error could not determine executable to run`
```
npx cap add android 실행 시 발생
```
**원인:** npx가 로컬 패키지를 찾지 못함  
**해결:** 전역 설치 후 직접 실행
```bash
npm install -g @capacitor/cli
cap add android
```

---

### 오류 2: `Missing appId / Could not find the android platform`
```
[error] The Capacitor CLI needs to run at the root of an npm package
```
**원인:** 터미널이 프로젝트 루트가 아닌 다른 폴더에 있음  
**해결:** 프로젝트 루트로 이동 후 재실행
```bash
cd C:\Users\lucky\Desktop\MyBob
cap add android
```

---

### 오류 3: `capacitor.settings.gradle does not exist`
```
Could not read script 'android\capacitor.settings.gradle' as it does not exist
```
**원인:** `cap sync`가 정상 실행되지 않아 파일 미생성  
**해결:** 루트에서 sync 재실행
```bash
node_modules/.bin/cap sync android
```
그래도 안 되면 파일 직접 생성:
```
android/capacitor.settings.gradle 내용:
// DO NOT EDIT THIS FILE
include ':capacitor-android'
project(':capacitor-android').projectDir = new File('../node_modules/@capacitor/android/capacitor')
```

---

### 오류 4: `capacitor.build.gradle does not exist`
```
Could not read script 'android\app\capacitor.build.gradle' as it does not exist
```
**원인:** cap sync 미완료로 파일 미생성  
**해결:** cap sync 재실행으로 자동 생성됨 (오류 3 해결 후 같이 해결됨)

---

### 오류 5: `Project :capacitor-android could not be found`
**원인:** `capacitor.settings.gradle`에 capacitor-android 모듈 경로 미등록  
**해결:** 오류 3과 동일, cap sync 재실행

---

### 오류 6: `No matching variant of project :capacitor-cordova-android-plugins`
**원인:** `android/capacitor-cordova-android-plugins/` 폴더 미생성  
**해결:** cap sync 재실행 (이 폴더도 sync 시 자동 생성됨)

---

### 오류 7: `error: invalid source release: 21`
```
Task :capacitor-android:compileDebugJavaWithJavac FAILED
Java compilation initialization error: invalid source release: 21
```
**원인:** Capacitor 8은 Java 21 이상 필요, 시스템이 Java 17을 사용 중  
**해결 1:** JAVA_HOME 환경변수를 Java 21+ 경로로 변경  
**해결 2 (환경변수 변경 후에도 안 될 때):** `android/gradle.properties`에 직접 지정
```properties
org.gradle.java.home=C:\\Program Files\\Eclipse Adoptium\\jdk-25.0.3.9-hotspot
```
변경 후 Gradle 데몬 재시작:
```bash
.\gradlew --stop
.\gradlew assembleDebug
```

## 8. APK 설치 방법

### 방법 A: USB 연결 (개발자 옵션 필요)
```bash
# 폰을 USB로 연결 후
adb install android\app\build\outputs\apk\debug\app-debug.apk
```

### 방법 B: 파일 전송
1. `app-debug.apk` 파일을 카카오톡/구글드라이브로 폰에 전송
2. 폰에서 파일 열기
3. "알 수 없는 출처 허용" 설정 → 설치

## 9. 다음 단계 (Play Store 출시)

### 릴리즈 APK 서명 (디버그 APK는 스토어 등록 불가)
```bash
# 1. keystore 생성 (1회만)
keytool -genkey -v -keystore mybob-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias mybob

# 2. android/app/build.gradle에 서명 설정 추가
# 3. .\gradlew assembleRelease 로 릴리즈 APK 빌드
```

### 스토어 등록 준비
- 앱 아이콘 교체 (현재 기본 Capacitor 아이콘)
- 스크린샷 준비 (폰 화면 캡처)
- 개인정보처리방침 URL (앱 내 페이지 또는 외부 URL)
- 앱 설명 (한국어)

### 구조 개선 (선택사항)
- 현재: 웹뷰로 Vercel URL 호출 (인터넷 필수)
- 개선: 프론트엔드를 앱에 번들링, API만 Vercel 호출 (오프라인 일부 지원)

## 10. 핵심 파일 구조 요약

```
MyBob/
├── capacitor.config.ts          # Capacitor 설정 (appId, Vercel URL 등)
├── package.json                 # cap:sync, cap:open 스크립트 포함
├── android/                     # cap add android로 자동 생성 (git 제외)
│   ├── capacitor.settings.gradle    # capacitor-android 모듈 경로
│   ├── gradle.properties            # Java 경로, JVM 설정
│   ├── app/
│   │   ├── capacitor.build.gradle   # 빌드 설정
│   │   ├── src/main/
│   │   │   ├── AndroidManifest.xml  # 앱 권한 설정
│   │   │   └── assets/
│   │   │       └── capacitor.config.json  # 런타임 설정 (sync 시 자동 생성)
│   │   └── build/outputs/apk/debug/
│   │       └── app-debug.apk        # 빌드된 APK
│   └── capacitor-cordova-android-plugins/  # sync 시 자동 생성
└── node_modules/@capacitor/android/       # Capacitor Android 코어
```

## 11. 자주 쓰는 명령어 요약

```bash
# 설정 변경 후 android/ 에 반영
node_modules/.bin/cap sync android

# Android Studio 열기
node_modules/.bin/cap open android

# APK 빌드 (android/ 폴더 안에서)
.\gradlew assembleDebug

# Gradle 데몬 재시작 (빌드 이상할 때)
.\gradlew --stop

# 폰에 APK 설치 (USB 연결 시)
adb install app\build\outputs\apk\debug\app-debug.apk
```
